The pipeline

Four stages between an email and your system of record

Every invoice that arrives in an InvoicePass inbox passes through the same deterministic pipeline. No queues you have to manage. No webhooks you have to retry. The protocol is the contract.

[01]
SMTP inbound
Authenticated mail accepted from any sender. MIME parsed, attachments extracted, headers preserved.
// MIME · DKIM · threading
[02]
PDF extract
Metadata pulled via XMP, then subject line, then sender heuristics. Fallback cascade resolves 99.2% of invoices.
// XMP · subject · heuristic
[03]
Three-signal dedup
SHA-256 hash, invoice number, and amount window cross-checked against your last 12 months. Duplicates flagged, not silently dropped.
// hash · number · amount
[04]
Signed delivery
A signed URL pushed to your endpoint. TLS-only, TTL one hour, replay-safe. Or stays in the portal for human review.
// webhook · TTL 1h · TLS
Integrate

One endpoint. Five lines of code

If you can receive a webhook, you can ingest invoices. If you can't, route to your portal inbox and triage there. No SDK required, no vendor lock.

POST /api/v1/inbox

Subscribe your endpoint. We deliver each invoice with a signed payload, a deterministic ID, and full audit trail. Idempotent by Message-ID — replay anytime.

  • Bearer auth, timing-safe comparison
  • Exponential backoff, 24h retry window
  • Quota: 50 PDFs / operator / month during beta
  • Storage in ca-central-1 · data residency on request 
# subscribe a webhook to receive parsed invoices
curl -X POST https://api.invoicepass.app/v1/inbox \
  -H "Authorization: Bearer ipk_live_7f3a..." \
  -H "Content-Type: application/json" \
  -d '{
    "endpoint": "https://api.acme.co/invoices",
    "events": ["invoice.received", "invoice.duplicate"],
    "format": "v1"
  }'

# → {"id": "sub_8ZqK...", "status": "active"}
import { InvoicePass } from "invoicepass";

const ip = new InvoicePass(process.env.IPK);

await ip.inbox.subscribe({
endpoint: "https://api.acme.co/invoices",
events: ["invoice.received", "invoice.duplicate"],
format: "v1"
});

// invoices arrive at /invoices, signed and deduped
from invoicepass import InvoicePass

ip = InvoicePass(api_key=os.getenv("IPK"))

ip.inbox.subscribe(
endpoint="https://api.acme.co/invoices",
events=["invoice.received", "invoice.duplicate"],
format="v1"
)

# signed, deduped, idempotent. that's it.
How it works

Three steps. No setup, no software

You don't install anything. You don't configure DNS. You pick a name, give it to your vendors, and the inbox starts working.

[01]
Pick your inbox name

Choose a short handle — for example silva@invoicepass.app. It's yours, it's permanent, it's free.

[02]
Send invoices to it

Give the address to vendors. Or forward existing invoices from Gmail, Outlook, anywhere. PDFs are parsed automatically.

[03]
Review them in your portal

Every invoice is signed, deduped, and organized by vendor. Export to your accountant when you're ready.
